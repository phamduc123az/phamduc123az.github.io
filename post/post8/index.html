DOM là gì? Tìm hiểu và thao tác DOM trong Javascript

 Posted on March 7, 2017  |  

DOM là gì?

DOM là tên gọi viết tắt của Document Object Model (tạm dịch: Mô hình Các Đối tượng Tài liệu). Đây là một chuẩn được định nghĩa bởi W3C (Tổ Chức Web Toàn Cầu – World Wide Web Consortium). DOM được sử dụng để truy xuất và thao tác trên các tài liệu có cấu trúc dạng HTML hay XML thông qua các ngôn ngữ lập trình phổ biến như Javascript, PHP, v.v.

Cây cấu trúc DOM

Node

Trong HTML DOM, mọi thành phần đều được xem là một node (nút), được biểu diễn trên một cây cấu trúc dạng cây gọi là DOM Tree. Các phần tử khác nhau sẽ được phân loại thành các node khác nhau, nhưng quan trọng nhất là ba loại:

  1. Node gốc: Chính là tài liệu HTML, thường được biểu diễn bởi thẻ <html>.
  2. Node phần tử: Biểu diễn cho một phần tử HTML.
  3. Node văn bản: Mỗi đoạn ký tự trong tài liệu HTML, bên trong một thẻ HTML, đều là một node văn bản. Ví dụ như tên trang web trong thẻ <title>, tên đề mục trong thẻ <h1>, hoặc một đoạn văn trong thẻ <p>.

Ngoài ra còn có các node khác như:

  • Node thuộc tính (attribute node)
  • Node chú thích (comment node)

Quan hệ giữa các node

  • Node gốc (document) luôn là node đầu tiên.
  • Tất cả các node không phải là node gốc đều chỉ có một node cha (parent).
  • Một node có thể có một hoặc nhiều con, nhưng cũng có thể không có con nào.
  • Những node có cùng node cha được gọi là các node anh em (siblings).
  • Trong các node anh em, node đầu tiên được gọi là con cả (firstChild) và node cuối cùng là con út (lastChild).

Các loại DOM trong Javascript

Javascript cung cấp nhiều loại DOM để xử lý HTML và CSS một cách dễ dàng hơn:

  1. DOM document: Lưu trữ toàn bộ các thành phần trong document của website.
  2. DOM element: Truy xuất tới thẻ HTML thông qua các thuộc tính như tên class, id, name của thẻ HTML.
  3. DOM HTML: Thay đổi giá trị nội dung và giá trị thuộc tính của các thẻ HTML.
  4. DOM CSS: Thay đổi các định dạng CSS của thẻ HTML.
  5. DOM Event: Gán các sự kiện như onclick(), onload() vào các thẻ HTML.
  6. DOM Listener: Lắng nghe các sự kiện tác động lên thẻ HTML.
  7. DOM Navigation: Dùng để quản lý, thao tác với các thẻ HTML, thể hiện mối quan hệ cha-con của các thẻ HTML.
  8. DOM Node, Nodelist: Thao tác với HTML thông qua đối tượng (Object).

Thao tác với DOM

Mọi nội dung đều có thể được cập nhật động thông qua các thuộc tính và phương thức của DOM. Bạn có thể thay đổi định dạng chữ, nội dung chữ, thay đổi cấu trúc các node, và thậm chí thêm các node mới.

Các Thuộc tính và Phương thức thường gặp

Thuộc tính:

  • id: Định danh – duy nhất cho mỗi phần tử, thường dùng để truy xuất DOM trực tiếp và nhanh chóng.
  • className: Tên lớp – dùng để truy xuất DOM như id, nhưng một className có thể dùng cho nhiều phần tử.
  • tagName: Tên thẻ HTML.
  • innerHTML: Trả về mã HTML bên trong phần tử hiện tại. Đây là chuỗi ký tự chứa tất cả phần tử bên trong, bao gồm các node phần tử và node văn bản.
  • outerHTML: Trả về mã HTML của phần tử hiện tại. Nói cách khác, outerHTML = tagName + innerHTML.
  • textContent: Trả về chuỗi ký tự chứa nội dung của tất cả node văn bản bên trong phần tử hiện tại.
  • attributes: Tập hợp các thuộc tính như id, name, class, href, title, v.v.
  • style: Tập hợp các định dạng của phần tử hiện tại.
  • value: Lấy giá trị của thành phần được chọn thành một biến.

Phương thức:

  • getElementById(id): Tham chiếu đến một node duy nhất có thuộc tính id giống với id cần tìm.
  • getElementsByTagName(tagname): Tham chiếu đến tất cả các node có thuộc tính tagName giống với tên thẻ cần tìm. Ví dụ, để truy xuất tất cả các thẻ trong tài liệu HTML, hãy sử dụng document.getElementsByTagName('*').
  • getElementsByName(name): Tham chiếu đến tất cả các node có thuộc tính name cần tìm.
  • getAttribute(attributeName): Lấy giá trị của thuộc tính.
  • setAttribute(attributeName, value): Sửa giá trị của thuộc tính.
  • appendChild(node): Thêm một node con vào node hiện tại.
  • removeChild(node): Xóa một node con khỏi node hiện tại.

Thuộc tính quan hệ:

Vì các phần tử DOM đều là các node trên cây cấu trúc DOM, chúng sở hữu các thuộc tính quan hệ để biểu diễn sự phụ thuộc giữa các node:

  • parentNode: Node cha.
  • childNodes: Các node con.
  • firstChild: Node con đầu tiên.
  • lastChild: Node con cuối cùng.
  • nextSibling: Node anh em liền kề sau.
  • previousSibling: Node anh em liền kề trước.
